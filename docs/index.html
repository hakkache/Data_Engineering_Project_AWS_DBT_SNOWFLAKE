Project Overview

Objective

Production-ready data engineering solution analyzing Brazilian E-Commerce data (2016-2018) from Kaggle/Olist, featuring automated data pipeline, transformation, and analytics.

Data Source

8 source tables with 100K+ orders from Brazilian e-commerce marketplace, stored in AWS S3 and processed through Snowflake data warehouse.

Processing

Medallion architecture (Bronze/Silver/Gold) with incremental loads, data quality tests, and business logic transformations using dbt.

Analytics

Interactive Streamlit dashboard with 7 pages covering sales, customer, product, payment, delivery, and review analytics.

Deployment Guide

1. Clone Repository

git clone https://github.com/hakkache/Data_Engineering_Project_AWS_DBT_SNOWFLAKE.git

2. Configure Snowflake

# Edit profiles.yml with your Snowflake credentials
account: your_account
user: your_username
password: your_password
warehouse: your_warehouse

3. Setup Data Warehouse

# Create schemas in Snowflake
CREATE SCHEMA RAW;
CREATE SCHEMA BRONZE;
CREATE SCHEMA SILVER;
CREATE SCHEMA GOLD;

4. Run DBT Models

cd aws_dbt_snowflake_project
dbt deps
dbt run
dbt test

5. Launch Dashboard

cd dashboard
pip install -r requirements.txt
streamlit run app.py

Important Notes

  • Ensure AWS S3 bucket is accessible with proper IAM roles
  • Snowflake warehouse should have adequate compute resources
  • Configure environment variables for sensitive credentials
  • Run dbt tests to validate data quality after each transformation
  • Monitor Snowflake credits usage for cost optimization
